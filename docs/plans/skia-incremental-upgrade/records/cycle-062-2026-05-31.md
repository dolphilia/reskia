## C API Catch-up

- updated: `SkImage_isValid` keeps `reskia_recording_context_t*` ABI and calls `SkImage::isValid(SkRecorder*)` with a null recorder for legacy compatibility
- updated: `SkImageGenerator_isValid` keeps `reskia_recording_context_t*` ABI and calls `SkImageGenerator::isValid(SkRecorder*)` with a null recorder for legacy compatibility
- updated: `SkImage_makeSubset`, `SkImage_makeColorSpace`, and `SkImage_makeColorTypeAndColorSpace` keep legacy direct-context C ABI and call recorder/property overloads with null recorder/default properties
- retained: explicit Graphite recorder C APIs continue to pass a real `SkRecorder*`
- rationale: deriving `SkRecorder*` from legacy Ganesh context wrappers would require pulling Ganesh internals into always-built C API code; null-recorder compatibility keeps the old ABI stable while preserving the explicit recorder-aware APIs
