## 既知の注意点

- `SkYUVAInfo_newWithDimensionsConfigSubsamplingSpaceOriginAndSiting` は `sk_i_size_t` handle を受け取る。handle `0` の扱いを決める必要がある。
- `SkEncodedOrigin` は有効範囲が `1..kLast_SkEncodedOrigin`。`0` は invalid として扱う方針が既存バッチと合う。
- enum static helper には `SkUNREACHABLE` に落ちるものがあるため、C API 側で enum 範囲を必ず確認する。
- `planeDimensions` や `planeSizes` のような output array は NULL 許容か必須かを upstream 実装ごとに確認してから規約化する。
- handle table の `*_get_entity(0)` は default object を返す実装が多いが、C API の入力検証では invalid handle を明示的に拒否する方が安全な場合がある。
